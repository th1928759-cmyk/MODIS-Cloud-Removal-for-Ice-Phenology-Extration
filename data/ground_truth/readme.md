# 贝加尔湖冰情物候提取：地面真值 (Ground Truth) 获取说明

本目录包含了用于验证 MODIS/MYD 去云算法及湖冰物候提取精度的地面真值（Ground Truth）数据集。

由于缺乏大范围、高频次的实地观测数据，本研究的真值数据是通过在 Google Earth Engine (GEE) 平台上，对**高分辨率 Sentinel-1 SAR（合成孔径雷达）影像进行人工目视解译**获取的。

## 1. 验证方法：为什么选择 Sentinel-1 SAR？

与光学传感器（如 MODIS, Landsat）极易受云层遮挡不同，Sentinel-1 C波段 SAR 具有**穿透云层、全天候、全天时**观测的能力。这使得它成为验证高纬度湖泊冰情（尤其是多云的封冻期和解冻期）最完美的“金标准”。

* **水体 (开阔水面)：** 表现为**暗色**。由于水面相对光滑，雷达波发生镜面反射，回波信号极弱。
* **湖冰 (冰盖/浮冰)：** 表现为**亮色/强纹理**。雷达波穿透冰层并在内部的裂隙、气泡中发生体散射，强烈的回波信号返回传感器。

## 2. 真值采样复现指南

为了提高目视解译的效率和严谨性，我们在 Google Earth Engine 中开发了一套定制化的交互式 UI 工具（见 `GEE_Sentinel1_Sampling.js`）。该脚本执行**时空均衡采样（Spatio-temporally Balanced Sampling）**，并强制进行几何求交，确保所有验证点严格落在湖泊边界内部。

### GEE 交互式验证工具使用流程：
1. **加载脚本：** 将 `GEE_Sentinel1_Sampling.js` 中的代码复制并粘贴至 [Google Earth Engine Code Editor](https://code.earthengine.google.com/)。
   > *注意：请在代码开头将 `baikalAssetPath` 替换为你个人上传的贝加尔湖 Shapefile 资产路径。*
2. **生成样本：** 点击左侧面板的 `▶️ 生成春季样本` 按钮。脚本将自动筛选与湖泊相交的影像并生成随机点。
3. **判读打标签：** 地图会自动缩放至目标像素周围 3km 的缓冲区。
   - 如果像素及其周围呈现明亮色块/明显纹理，点击 **🧊 冰 (1)**。
   - 如果像素区域呈现暗黑色、平滑无纹理，点击 **💧 水 (0)**。
   - 如果影像受强风引起的表面粗糙度干扰难以判读，点击 **❓ 跳过/未知 (-1)**。
4. **导出数据：** 所有点位判读完毕后，点击 `📥 导出CSV` 将带有真值标签的数据保存至 Google Drive，并存入本项目的 `raw_exports/` 目录中。

## 3. 后期处理 (Post-Processing)
导出的原始 CSV 文件包含的经纬度为 WGS84 坐标系。在运行精度验证脚本之前，需要将这些坐标投影转换至与 MODIS 数据一致的 **EPSG:32648** 坐标系，以确保后续提取的像元位置严格对齐。最终合并的处理文件即为本目录下的 `S1_GroundTruth_Projected.csv`。
